# 🎯 AI 집중도 분석 시스템 개발 현황 보고




### 1. **백엔드 시스템** (`edtech-backend/`)
- **기술스택**: Spring Boot + Java + MySQL
- **주요 기능**:
  - 사용자별 학습 데이터 관리
  - 집중도 분석 API 제공
  - 대시보드 KPI 데이터 산출
  - 실시간 데이터 처리

### 2. **AI 분석 서버** (`fastapi-service/`)
- **기술스택**: FastAPI + Python + TensorFlow
- **주요 기능**:
  - 실시간 감정 분석 (FER2013 모델)
  - 시선 추적 분석
  - 집중도 스코어 계산
  - 웹캠 기반 실시간 분석자

### 3. **프론트엔드** (`edtech-frontend/`)
- **기술스택**: React + Tailwind CSS
- **주요 기능**:
  - 사용자별 맞춤 대시보드
  - 실시간 차트 및 그래프
  - 직관적인 KPI 표시
  - 반응형 UI/UX

## 🚀 핵심 구현 기능

### ✅ 완료된 주요 기능들

#### **🚨 NEW! 실시간 집중도 경고 알림 시스템**
- **스마트 모니터링**: 5초마다 집중도 자동 체크
- **맞춤형 알림**: 30% 이하 경고, 15% 이하 긴급 알림
- **시간대별 조언**: 오전/오후/저녁 맞춤 메시지
- **브라우저 알림**: 실제 알림창 팝업 + 시각적 효과
- **개선 가이드**: 구체적인 집중도 향상 방법 제시
- **쿨다운 시스템**: 30초 간격으로 스팸 방지

#### 1. **사용자별 차별화된 대시보드**
- 4명의 서로 다른 학습 패턴을 가진 사용자 데이터
- 개인별 맞춤 KPI (출석률, 복습률, 집중도)
- URL 파라미터 및 드롭다운을 통한 사용자 전환

#### 2. **실시간 AI 분석 시스템**
- 웹캠을 통한 실시간 감정 분석
- 시선 추적을 통한 집중도 측정
- 머신러닝 모델 기반 점수 산출

#### 3. **고도화된 데이터 시각화**
- 시간대별 집중도 변화 차트
- 인터랙티브 툴팁 및 호버 효과
- 주간 변화율 트렌드 분석

#### 4. **REST API 시스템**
- 사용자별 데이터 조회 API
- 실시간 분석 결과 API
- 통계 및 리포트 생성 API

## 🎯 시뮬레이션 시스템 구조

### **실시간 집중도 분석 플로우**
```
👤 사용자 → 📸 웹캠 → 🤖 AI 분석 → 📊 집중도 점수 → ⚠️ 알림 시스템
```

### **시뮬레이션 vs 실제 운영**

| 구분 | 시뮬레이션 모드 (현재) | 실제 운영 모드 |
|------|------------------------|---------------|
| **데이터 소스** | 수학적 시뮬레이션 | 웹캠 + AI 분석 |
| **집중도 계산** | 사용자별 베이스 점수 + 시간대 변동 + 랜덤 | 실제 감정/시선/자세 분석 |
| **업데이트 주기** | 5초마다 자동 | 5초마다 실시간 분석 |
| **알림 시스템** | ✅ 완전 구현 | ✅ 완전 구현 |
| **사용자 경험** | ✅ 동일한 UX | ✅ 동일한 UX |

### **시뮬레이션 데이터 생성 방식**
- **사용자 1**: 45% 베이스 (기본형) → 오후 2PM 피크
- **사용자 2**: 65% 베이스 (우수형) → 오전 집중도 높음
- **사용자 3**: 35% 베이스 (저조형) → 저녁에도 집중도 유지
- **사용자 4**: 55% 베이스 (균형형) → 전반적으로 안정적

### **알림 메시지 예시**
- **경고 (30% 이하)**: "⚠️ 집중도 저하 감지 (25%) - 바른 자세로 앉아보세요"
- **긴급 (15% 이하)**: "🚨 집중도 위험 수준 (12%) - 즉시 5-10분 휴식을 권장합니다"
- **시간대별**: "🚨 오후 집중력 한계 (18%) - 점심 후 졸음, 10분 파워냅을 고려하세요"

## 🎮 데모 실행 방법

### 1단계: 시스템 구동
```bash
# MySQL 데이터베이스 실행
docker-compose up -d

# Spring Boot 백엔드 실행
cd edtech-backend && ./gradlew bootRun

# FastAPI AI 서버 실행
cd fastapi-service && python -m uvicorn app.main:app --port 8001

# React 프론트엔드 실행
cd edtech-frontend && npm start
```

### 2단계: 데모 화면 접속
- **메인 대시보드 (사용자 1)**: http://localhost:3000/new-dashboard
- **우수 학습자 (사용자 2)**: http://localhost:3000/new-dashboard?userId=2
- **성실형 학습자 (사용자 3)**: http://localhost:3000/new-dashboard?userId=3
- **균형형 학습자 (사용자 4)**: http://localhost:3000/new-dashboard?userId=4
- **실시간 카메라 분석**: http://localhost:3000/camera
- **데이터 플로우 데모**: http://localhost:3000/demo

### 3단계: 집중도 알림 시스템 체험
- **브라우저 알림 권한 허용** (중요!)
- **실시간 모니터링** 섹션에서 집중도 변화 관찰
- **경고 임계값을 50%로 조정** (더 자주 알림 받기 위해)
- **5초 기다리기** → 자동으로 경고 알림 팝업!
- **집중도 개선 가이드** 자동 표시 확인

### 4단계: 사용자별 데이터 확인
- 우상단 사용자 선택 드롭다운 사용
- 차트 영역에 마우스 호버로 상세 정보 확인
- KPI 카드 수치 변화 실시간 확인
